# Marpスライド生成用汎用プロンプト

以下のプロンプトを使用して、テキスト情報からMarpスライドを生成してください。

## プロンプトテンプレート

```
以下のテキスト情報を基に、Marpスライドを作成してください。
sample-slide.mdを参考にして、適切なレイアウトとクラスを使用してください。

**重要**: マークダウンの可読性を保つため、HTMLタグは極力使わず、マークダウン記法を優先してください。
レイアウト変更は`<!-- _class: [クラス名] -->`のコメントで行い、HTMLとCSSを一緒に書かないでください。

【テキスト情報】
[ここにテキスト情報を貼り付けてください]

【指示】
**重要**: レイアウト選択時は、以下の「レイアウト判定表」と「内容別レイアウト選択基準」を必ず参照して、内容に最も適したレイアウトを選択してください。

**最優先原則**: 内容を相手に効果的に伝えることを最優先に、以下の基準に基づいて最適なスライドを選定してください。
- **重要**: 図・画像は一切使用しない。テキストのみで内容を伝える
- テキストのみで十分伝わる内容は、画像レイアウトを使わない
- 視覚的な説明が必要な場合でも、図・画像は使用せず、テキストのみで説明する
- 画像URLが提供されていても、図・画像は使用しない

**マークダウンの可読性を保つ（重要）**：
- **HTMLは極力書かない**: マークダウンの最大のメリットは可読性とシンプルさです。HTMLを多用すると可読性が下がり、AIが適切なレイアウトを選択できなくなります
- **クラス指定はスライド上部のコメントのみ**: レイアウト変更は`<!-- _class: [クラス名] -->`のコメントで行い、HTMLタグは最小限に
- **マークダウン記法を優先**: 見出し、リスト、コードブロック、テーブルなどはマークダウン記法を使用
- **カスタムCSSは必要な場合のみ**: ステップ・フロー説明用レイアウトなど、特別な場合のみカスタムCSSを使用

1. スライドの構造：
   - フロントマターに以下を必ず含める：
     - marp: true
     - theme: sunaneko
     - paginate: true
     - title: [スライドのタイトル]
     - description: [スライドの説明]
     - **マイルストーンスライドを含む場合**: `html: true`を追加（必須）
   - **コードブロックの自動スケーリング**: `sunaneko`テーマには既に`@auto-scaling true`が含まれているため、フロントマターには追加しない（追加するとCSSが正しく適用されない可能性がある）

2. 最初のスライドはタイトルページとして作成：
   - `<!-- _class: title -->`
   - `<!-- _paginate: false -->`
   - **重要**: CSSはスライド内に書かない。`sunaneko-theme.css`に既に定義されている（h1の改行防止も含む）
   - メインタイトル（h1）
   - サブタイトル（h2、必要に応じて）
   - **ロゴ画像を必ず含める**: `![sunaneko-logo w:400px](https://raw.githubusercontent.com/suh-sunaneko/sunaneko-marp-slide/main/sunaneko_inversion_logo.png)`
   - 日付や執筆者情報は含めない（タイトルページはシンプルに）

3. 目次ページ（アジェンダ）には以下を使用：
   - `<!-- _class: align-center agenda -->`
   - `<!-- _paginate: false -->`
   - 目次が8つ以上の場合 → `<!-- _class: align-center agenda column-layout -->` を使用して2カラム表示
   - **重要**: 8つ以上の場合、1-7を左カラム、8つ目からを右カラムに表示する
   - テーブル形式で目次を表示（例: `| 01 | アジェンダ名 |`）

4. セクション区切りには以下を使用：
   - `<!-- _class: section -->`
   - `<!-- _paginate: false -->`
   - **セクションタイトルは「セクション名 | 説明」の形式**: 例: `## 1. プロジェクト概要 | プロジェクトの基本情報`
   - セクションページは背景色がグレーになり、テキストは左寄せの中央に配置される

5. レイアウトの選択指針（内容に応じた判定基準）：
   
   **判定フロー（各スライドごとに実行）**：
   
   ```
   Step 0: 画像の必要性を判定（重要：闇雲に画像を使わない）
   ├─ テキスト情報に画像URLが明示的に含まれている？
   │  └─ YES → 提供された画像URLを使用して画像レイアウトを検討
   ├─ 視覚的な説明が必須か？（アーキテクチャ図、フローチャート、グラフ、図解など）
   │  ├─ YES → プレースホルダー画像を使用して画像レイアウトを検討
   │  │   └─ プレースホルダー画像URL: https://placehold.jp/ffffff/8c8c8c/300x200.png
   │  └─ NO → 次の判定へ
   ├─ テキストのみで十分伝わる内容か？（概念説明、リスト、手順など）
   │  └─ YES → 基本レイアウト（画像不要）
   └─ 画像があると理解が深まるが、なくても伝わる内容か？
      └─ テキストのみのレイアウトを優先（画像は不要）
   
   Step 1: 内容の種類を判定
   ├─ テキストのみ？
   ├─ 画像が含まれる？（Step 0で判定済み）
   ├─ コードブロックが含まれる？
   ├─ テーブルが含まれる？
   └─ リスト・箇条書きが多い？
   
   Step 2: 情報量を確認
   ├─ 少ない（3-5行、3-5項目）
   ├─ 中程度（6-10行、6-10項目）
   └─ 多い（11行以上、11項目以上）
   **重要**: 実際に行数を数えてから判断すること。リスト項目が6個程度、テキストが10行程度の場合は1枚に収まる可能性が高い。不必要に分割しないこと
   
   Step 3: 目的・構造を確認
   ├─ 通常の説明？
   ├─ 強調・インパクト重視？
   ├─ 比較・対比？
   ├─ ステップ・フロー説明？（各ステップにタイトルと説明文がある）
   └─ フルスクリーン表示？
   
   Step 4: 判定表を参照して適切なレイアウトを選択
   └─ 複数の条件に該当する場合は、最も重要度の高い条件を優先
   └─ 画像の必要性が不明確な場合は、テキストのみのレイアウトを優先
   ```
   
   **画像の必要性判定基準（重要）**：
   
   **重要**: 図・画像は一切使用しない。すべての内容はテキストのみで表現する。
   
   **画像レイアウトは使用しない**：
   - ❌ テキスト情報に画像URLが明示的に含まれていても、画像は使用しない
   - ❌ アーキテクチャ図、システム構成図など視覚的な説明が必要な場合でも、テキストのみで説明する
   - ❌ UI画面、スクリーンショットなど実際の画面を見せる必要がある場合でも、テキストのみで説明する
   - ❌ フローチャート、図表など図解が必要な場合でも、テキストのみで説明する
   - ❌ グラフ、チャートなどデータの可視化が必要な場合でも、テキストのみで説明する
   - ❌ 比較のために複数の画像を並べる必要がある場合でも、テキストのみで説明する
   - ❌ すべての内容はテキストのみで表現する
   
   **内容別レイアウト選択基準**：
   
   - **テキストのみの場合**：
     - 情報量が少ない（3-5行程度）→ 基本レイアウト（クラス指定なし）
     - 情報量が多い（6行以上）→ `<!-- _class: small-text -->` を追加検討
     - 強調したい・目立たせたい → `<!-- _class: all-text-center align-center -->`
     - 見出しのみ・タイトル的な内容 → `<!-- _class: all-text-center -->`
   
   - **画像が含まれる場合**：
     - **重要**: 図・画像は一切使用しない。すべての内容はテキストのみで表現する
   
   - **コードブロックが含まれる場合**：
     - コードが短い（10行以下）→ 基本レイアウト
     - コードが長い（11行以上）→ `<!-- _class: small-text -->` を検討
     - コード+説明文 → 基本レイアウト（コードブロックは自動でサイズ調整される）
   
   - **比較・対比・並列の内容**：
     - 3つの項目を比較 → `<!-- _class: column-layout -->` と `<div class="column">` を使用
     - 2つの項目を比較 → 基本レイアウトで箇条書きまたはテーブル
     - **数値データ・統計情報がある場合** → 基本レイアウトでテキストのみで説明（グラフやチャートは使用しない）
     - **組織構造・体制図がある場合** → 基本レイアウトでテキストのみで説明（組織図は使用しない）
     - **現状と改善後の比較がある場合** → 基本レイアウトでテキストのみで説明（比較図は使用しない）
     - 手順・プロセス → 基本レイアウトで番号付きリスト
   
   - **ステップ・フロー説明の場合**：
     - ステップ、フェーズ、フロー、プロジェクトフェーズなどを視覚的に説明する場合 → `<!-- _class: column-layout -->` を使用
     - 各ステップにタイトル（h2）と説明文がある場合に最適
     - h2のスタイルはテーマで既に設定されている（背景色、パディング、角丸など）
     - **ステップの表記**: ステップは「STEP1」「STEP2」のように「STEP」を使用する
     - **カラム数の調整**：
       - 2〜3つのステップ: カスタムCSS不要（テーマのデフォルト設定で対応）
       - 4つのステップ: **CSSはスライド内に書かない**。`sunaneko-theme.css`に既に定義されている（4カラムレイアウトとh2の高さ統一も含む）。**重要**: 4つのステップの場合、h2のタイトルは6文字以内にする（改行を防ぐため）
       - 5つ以上のステップ: 複数のスライドに分割するか、カラム幅をさらに調整
   
   - **リスト・箇条書きが多い場合**：
     - 項目数が少ない（3-5個）→ 基本レイアウト
     - 項目数が多い（6個以上）→ `<!-- _class: small-text -->` を検討、または複数スライドに分割
   
   - **図表・テーブルが含まれる場合**：
     - テーブルが小さい（3-4行）→ 基本レイアウト
     - テーブルが大きい（5行以上）→ **必ず** `<!-- _class: table-custom -->` を使用（カスタムテーブルスタイルで見やすく表示）
     - テーブルを強調・見やすくしたい場合も `<!-- _class: table-custom -->` を使用
     - 情報量が非常に多い場合は `<!-- _class: table-custom small-text -->` を検討
     - **重要**: 図表・画像は一切使用しない。テキストのみで表現する
   
   - **マイルストーン・プロジェクトスケジュールの場合**：
     - マイルストーンやプロジェクトスケジュールを表示 → Mermaid Ganttチャート（基本レイアウト）
     - フロントマターにMermaidのJavaScriptライブラリを読み込むスクリプトを追加（必須）
     - **フェーズ名はPMBOKに合わせる**: 計画、実行、監視制御などを使用する（「フェーズ」という文字は付けない）
     - **重要**: ガントチャートのセクション名は6文字以内にする（例: 「監視制御」は4文字でOK、「計画フェーズ」は「計画」にする）
     - Ganttチャートの定義に`axisFormat %Y-%m`と`tickInterval 1month`を指定
     - 各作業に適切な期間を設定（例: `30d`, `28d`など）
     - その他のCSSは`sunaneko-theme.css`に統一されているため、Ganttチャートの横幅調整用CSS以外は追加しない
   
   - **強調・インパクト重視の場合**：
     - フルスクリーン表示 → `<!-- _class: no-header -->`
     - 中央揃えで目立たせる → `<!-- _class: all-text-center align-center -->`
     - **重要**: 図・画像は一切使用しない。テキストのみで表現する
   
   **重要**: 図・画像レイアウト（`content-image-right`, `content-image-left`）は使用しない

6. テキストの装飾：
   - 見出し内で**太字**にすると赤色のアクセントカラーになる
   - 通常のマークダウン記法（太字、斜体、リスト、コードブロック、テーブルなど）を活用
   - コードブロックは言語指定（例：```ts, ```python, ```bash など）
   - **コードブロックの自動スケーリング**: `sunaneko`テーマには自動スケーリング機能が含まれており、コードの大きさに合わせてコードブロック内のテキストが自動で小さくなります

7. 画像の使用：
   - **重要**: 図・画像は一切使用しない。すべての内容はテキストのみで表現する

8. ページネーション制御：
   - タイトルページ、目次ページ、セクションページでは `<!-- _paginate: false -->` を使用

9. スライド区切り：
   - 各スライドは `---` で区切る

10. その他の注意事項：
   - 1スライドに収まりきらない情報は複数のスライドに分割
   - **スライドの行数制限**: 1スライドあたり12行を上限とする（タイトル行、空行、リスト項目を含む）。12行を超える場合は必ず複数のスライドに分割する
   - **重要**: スライドを分割する前に、実際に行数を数えてから判断すること。リスト項目が6個程度、テキストが10行程度の場合は1枚に収まる可能性が高い。不必要に分割しないこと
   - 見出しは必ずh1（#）から始める
   - **マークダウンの可読性を最優先**: 
     - HTMLタグは極力使わない（`<div>`、`<span>`などは最小限に）
     - レイアウト変更は`<!-- _class: [クラス名] -->`のコメントで行う
     - マークダウン記法（見出し、リスト、コードブロック、テーブルなど）を優先して使用
     - **重要**: CSSはスライド内に書かない。すべてのCSSは`sunaneko-theme.css`に定義されている
   - **画像の使用**: 
     - **重要**: 図・画像は一切使用しない。すべての内容はテキストのみで表現する
     - 画像URLが提供されていても、画像は使用しない
     - 視覚的な説明が必要な場合でも、テキストのみで説明する
   - **ステップ・フロー説明用レイアウト**：
     - プロジェクトフェーズ、ステップ、フローなどを視覚的に説明する場合は、`<!-- _class: column-layout -->` を使用
     - **h2のスタイル**: h2のスタイル（背景色、パディング、角丸など）は`sunaneko-theme.css`に既に定義されているため、追加のCSSは不要
     - **カラム数の調整**：
       - 2〜3つのステップ: カスタムCSS不要
       - 4つのステップ: **CSSはスライド内に書かない**。`sunaneko-theme.css`に既に定義されている（4カラムレイアウトとh2の高さ統一も含む）。**重要**: 4つのステップの場合、h2のタイトルは6文字以内にする（改行を防ぐため）
     - 各ステップは`<div class="column">`で囲み、h2でタイトルを付ける
     - 使用例（4つのステップの場合）：
       ```markdown
       <!-- _class: column-layout -->
       
       # プロジェクトフェーズ
       
       <div class="column">
       
       ## STEP1
       現状分析・要件定義
       1月6日 - 3月31日（12週間）
       - 現状業務プロセスの分析と改善要件の定義
       
       </div>
       
       <div class="column">
       
       ## STEP2
       設計・開発
       4月1日 - 7月31日（17週間）
       - 新業務プロセスの設計とシステム開発
       
       </div>
       
       <div class="column">
       
       ## STEP3
       テスト準備
       8月1日 - 8月31日（4週間）
       - システムテストと導入準備
       
       </div>
       
       <div class="column">
       
       ## STEP4
       導入移行
       9月1日 - 9月30日（4週間）
       - 新システムの導入と業務移行
       
       </div>
       ```
   - 画像を使用する場合は、サイズを `![w:800px](画像URL)` のように指定
   - 情報量が多い場合は `small-text` クラスを検討
   - 内容を効果的に伝えることを最優先に、適切なレイアウトを選択
   - **マイルストーンスライド（Mermaid Ganttチャート）**：
     - マイルストーンやプロジェクトスケジュールを表示する場合は、MermaidのGanttチャートを使用
     - **フロントマターに`html: true`を追加**（必須）
     - **Ganttチャートのスライドの直前に以下を追加**（必須）：
       1. **スライドタイトル（h1見出し）**: Ganttチャートのスライドには必ずh1の見出しを追加する（例: `# プロジェクトスケジュール`）
       2. **重要**: CSSはスライド内に書かない。Ganttチャートの横幅・高さ調整は`sunaneko-theme.css`に既に定義されている
       3. **MermaidのJavaScriptライブラリを読み込むスクリプト**（スクリプトは非表示にする）：
          ```html
          <div class="mermaid-script" style="display:none;position:absolute;left:-9999px;width:0;height:0;overflow:hidden;">
          <script src="https://cdn.jsdelivr.net/npm/mermaid@10/dist/mermaid.min.js"></script>
          <script>
            mermaid.initialize({ 
              startOnLoad: true,
              gantt: {
                useWidth: 1800,
                useMaxWidth: true,
                leftPadding: 50,
                gridLineStartPadding: 25,
                fontSize: 10,
                sectionFontSize: 20,
                numberSectionStyles: 4,
                axisFormat: '%Y-%m',
                tickInterval: '1month',
                topPadding: 20,
                bottomPadding: 15,
                rightPadding: 10
              }
            });
          </script>
          </div>
          ```
     - **Ganttチャートの定義方法**（スライドタイトルを含む完全な例）：
       ```markdown
       ---
       
       # プロジェクトスケジュール
       
       <div class="mermaid-script" style="display:none;position:absolute;left:-9999px;width:0;height:0;overflow:hidden;">
       <script src="https://cdn.jsdelivr.net/npm/mermaid@10/dist/mermaid.min.js"></script>
       <script>
         mermaid.initialize({ 
           startOnLoad: true,
           gantt: {
             useWidth: 1280,
             useMaxWidth: true,
             leftPadding: 50,
             gridLineStartPadding: 25,
             fontSize: 10,
             sectionFontSize: 20,
             numberSectionStyles: 4,
             axisFormat: '%Y-%m',
             tickInterval: '1month',
             topPadding: 20,
             bottomPadding: 15,
             rightPadding: 10
           }
         });
       </script>
       </div>
       
       <div class="mermaid">
       gantt
           title プロジェクトマイルストーン
           dateFormat YYYY-MM-DD
           axisFormat %Y-%m
           tickInterval 1month
           section 計画
           要件定義: 2024-01-01, 30d
           基本設計: 2024-02-01, 28d
           section 実行
           詳細設計: 2024-03-01, 31d
           開発: 2024-04-01, 30d
       </div>
       ```
     - **重要な設定**：
       - `axisFormat %Y-%m`: 横軸を年月のみで表示（例: 2024-01, 2024-02）
       - `tickInterval 1month`: 月単位で目盛りを表示
       - `dateFormat YYYY-MM-DD`: 日付フォーマットを指定
       - 各作業には適切な期間を設定（例: `30d`, `28d`など。`0d`は使用しない）
     - **スクリプトの非表示**: スクリプトは`<div class="mermaid-script" style="display:none;position:absolute;left:-9999px;width:0;height:0;overflow:hidden;">`で囲み、スライドに表示されないようにする
     - **重要**: CSSはスライド内に書かない。Ganttチャートの横幅・高さ調整は`sunaneko-theme.css`に既に定義されている（上下左右にはみ出さないようにする設定も含む）
     - **現在日を示すライン**: 現在日を示す赤いラインは`sunaneko-theme.css`で自動的に非表示になる
     - **レイアウト**: 通常は基本レイアウト（クラス指定なし）を使用。Ganttチャートは自動的に縦中央に配置される

【出力形式】
完全なMarpマークダウンファイルとして出力してください。
```

## レイアウト判定表（クイックリファレンス）

| 内容の種類 | 画像URL | 情報量 | 目的 | 推奨レイアウト |
|---------|--------|--------|------|--------------|
| テキストのみ | なし | 少ない（3-5行） | 通常の説明 | 基本レイアウト（クラス指定なし） |
| テキストのみ | なし | 多い（6行以上） | 詳細説明 | `small-text` を追加 |
| テキストのみ | なし | 少ない | 強調・インパクト | `all-text-center align-center` |
| コードブロック | なし | 短い（10行以下） | コード提示 | 基本レイアウト |
| コードブロック | なし | 長い（11行以上） | コード提示 | `small-text` を追加 |
| 3項目の比較 | なし | - | 比較・対比 | `column-layout` |
| 2項目の比較 | なし | - | 比較・対比 | 基本レイアウトで箇条書きまたはテーブル |
| 数値データ・統計情報 | なし | - | データ可視化 | 基本レイアウトでテキストのみで説明 |
| 組織構造・体制図 | なし | - | 組織図表示 | 基本レイアウトでテキストのみで説明 |
| 現状と改善後の比較 | なし | - | 比較・対比 | 基本レイアウトでテキストのみで説明 |
| ステップ・フロー説明・プロジェクトフェーズ | なし | 2-3ステップ | 手順・プロセス説明 | `column-layout`（カスタムCSS不要） |
| ステップ・フロー説明・プロジェクトフェーズ | なし | 4ステップ | 手順・プロセス説明 | `column-layout` + カスタムCSS（カラム幅23%） |
| リスト・箇条書き | なし | 少ない（3-5個） | 項目列挙 | 基本レイアウト |
| リスト・箇条書き | なし | 多い（6個以上） | 項目列挙 | `small-text` または複数スライドに分割 |
| テーブル | なし | 小さい（3-4行） | データ表示 | 基本レイアウト |
| テーブル | なし | 大きい（5行以上） | データ表示 | **必ず** `table-custom` を使用（情報量が多い場合は `table-custom small-text` を検討） |
| 目次・アジェンダ | なし | 少ない（7個以下） | 目次表示 | `align-center agenda` |
| 目次・アジェンダ | なし | 多い（8個以上） | 目次表示 | `align-center agenda column-layout` |
| フルスクリーン表示 | なし/あり | - | 強調 | `no-header` |
| マイルストーン・スケジュール | なし | - | プロジェクトスケジュール表示 | Mermaid Ganttチャート（基本レイアウト） |

**注意**: 
- 画像URLが提供されていない場合でも、視覚的な説明が必須な場合（グラフ、図解、アーキテクチャ図など）は、プレースホルダー画像（`https://placehold.jp/ffffff/8c8c8c/300x200.png`）を使用して画像レイアウトを選択してください
- テキストのみで十分伝わる内容は、画像レイアウトを使わずにテキストのみのレイアウトを選択してください

## 利用可能なレイアウトクラス一覧

### 基本レイアウト
- クラス指定なし: 通常のテキスト中心のスライド

### 特殊レイアウト
- `title`: タイトルページ
- `agenda`: 目次ページ（アジェンダ）
  - `align-center agenda`: 目次を中央揃えで表示
  - `align-center agenda column-layout`: 目次が8つ以上の場合、2カラム表示（1-7を左カラム、8つ目からを右カラム）
- `section`: セクションページ（中扉）
- `no-header`: ヘッダーなし
- **重要**: 図・画像レイアウト（`image`, `image-shadow`, `content-image`, `content-image-right`, `content-image-left`）は使用しない
- `column-layout`: 横並びレイアウト（3カラム）
  - 通常のカラムレイアウト: 3つの項目を横並びに表示
  - ステップ・フロー説明用: カスタムCSSでh2のスタイルを変更して、ステップやフローを視覚的に説明
  - 目次が8つ以上の場合: `agenda`と組み合わせて2カラム表示（1-7を左カラム、8つ目からを右カラム）
- `table-custom`: カスタムテーブルスタイル（テーブルを強調・見やすく表示）

### テキスト配置
- `all-text-center`: すべてのテキストを中央揃え
- `text-center`: 段落テキストのみ中央揃え
- `h1-text-center` 〜 `h6-text-center`: 各見出しレベルを中央揃え
- `align-center`: スライド全体のテキストを縦方向中央揃え

### テキスト色
- `all-text-red`: すべてのテキストを赤色
- `h1-text-red` 〜 `h6-text-red`: 各見出しレベルを赤色
- `text-red`: 段落テキストのみ赤色
- `text-blue`: 段落テキストのみ青色

### その他
- `small-text`: フォントサイズを20%縮小
- `content-30`, `content-40`, `content-60`, `content-70`, `content-80`: テキスト領域の幅調整

## 注意事項

1. **フロントマターは必須**: 必ずスライドの先頭にフロントマターを含める
   - `sunaneko`テーマには既に`@auto-scaling true`が含まれているため、フロントマターには`@auto-scaling true`を追加しない（追加するとCSSが正しく適用されない）
2. **スライド区切り**: 各スライドは `---` で明確に区切る
3. **見出しの階層**: スライドタイトルは必ずh1（#）を使用
4. **レイアウトクラス**: `<!-- _class: [クラス名] -->` の形式で指定
5. **ページネーション**: タイトルページ、目次ページ、セクションページでは `<!-- _paginate: false -->` を使用
6. **目次ページ（アジェンダ）のデザイン**:
   - 目次が7個以下の場合 → `<!-- _class: align-center agenda -->` を使用
   - 目次が8個以上の場合 → `<!-- _class: align-center agenda column-layout -->` を使用して2カラム表示
   - **重要**: 8つ以上の場合、1-7を左カラム、8つ目からを右カラムに表示する
   - テーブル形式で目次を表示（例: `| 01 | アジェンダ名 |`）
7. **タイトルページのデザイン**: 
   - タイトルページには必ずロゴ画像を含める: `![sunaneko-logo w:400px](https://raw.githubusercontent.com/suh-sunaneko/sunaneko-marp-slide/mainsunaneko_inversion_logo.png)`
   - タイトルページはシンプルに、日付や執筆者情報は含めない
   - **h1の改行防止**: タイトルスライドのh1の改行防止は`sunaneko-theme.css`に既に定義されているため、スライド内にCSSを追加する必要はない
8. **最後のスライドのデザイン**: 
   - 最後のスライドは以下の形式を使用する（「ご清聴ありがとうございました」などのテキストは不要）:
     ```markdown
     <!-- _class: all-text-center align-center -->
     
     ![w:450px](https://raw.githubusercontent.com/suh-sunaneko/sunaneko-marp-slide/mainsunaneko_logo.png)
     ```
9. **セクションページのデザイン**: 
   - セクションタイトルは「セクション名 | 説明」の形式で記述: 例: `## 1. プロジェクト概要 | プロジェクトの基本情報`
   - セクションページは背景色がグレーになり、テキストは左寄せの中央に配置される
10. **マークダウンの可読性を保つ（重要）**: 
   - HTMLタグは極力使わない（`<div>`、`<span>`などは最小限に）
   - レイアウト変更は`<!-- _class: [クラス名] -->`のコメントで行う
   - マークダウン記法（見出し、リスト、コードブロック、テーブルなど）を優先して使用
   - HTMLとCSSを一緒に書かせない（カスタムCSSは必要な場合のみ）
   - マークダウンの可読性が下がると、AIが適切なレイアウトを選択できなくなる
11. **画像の使用（最重要）**: 
   - **重要**: 図・画像は一切使用しない。すべての内容はテキストのみで表現する
   - 画像URLが明示的に提供されていても、画像は使用しない
   - 視覚的な説明が必須な場合（グラフ、図解、アーキテクチャ図など）でも、テキストのみで説明する
   - すべての内容はテキストのみで表現する
12. **コードブロックの自動スケーリング**: `sunaneko`テーマには自動スケーリング機能が含まれており、コードの大きさに合わせてコードブロック内のテキストが自動で小さくなります
13. **情報量**: 1スライドに詰め込みすぎないよう、適切に分割（12行を上限とする）
14. **マイルストーンスライド（Mermaid Ganttチャート）**：
    - マイルストーンやプロジェクトスケジュールを表示する場合は、必ずMermaidのGanttチャートを使用
    - **フロントマターに`html: true`を追加**（必須）
    - **Ganttチャートのスライドの直前に以下を追加**（必須）：
      1. **スライドタイトル（h1見出し）**: Ganttチャートのスライドには必ずh1の見出しを追加する（例: `# プロジェクトスケジュール`）
      2. **重要**: CSSはスライド内に書かない。Ganttチャートの横幅・高さ調整は`sunaneko-theme.css`に既に定義されている
      3. **MermaidのJavaScriptライブラリを読み込むスクリプト**（スクリプトは非表示にする）：
         ```html
         <div class="mermaid-script" style="display:none;position:absolute;left:-9999px;width:0;height:0;overflow:hidden;">
         <script src="https://cdn.jsdelivr.net/npm/mermaid@10/dist/mermaid.min.js"></script>
         <script>
           mermaid.initialize({ 
             startOnLoad: true,
             gantt: {
               useWidth: 1920,
               useMaxWidth: true,
               leftPadding: 75,
               gridLineStartPadding: 35,
               fontSize: 11,
               sectionFontSize: 25,
               numberSectionStyles: 4,
               axisFormat: '%Y-%m',
               tickInterval: '1month',
               topPadding: 30,
               bottomPadding: 20,
               rightPadding: 10
             }
           });
         </script>
         </div>
         ```
    - Ganttチャートの定義には必ず`axisFormat %Y-%m`と`tickInterval 1month`を指定して、年月のみで表示する
    - 各作業には適切な期間を設定（`0d`ではなく、実際の期間を指定。例: `30d`, `28d`など）
    - **スクリプトの非表示**: スクリプトは`<div class="mermaid-script" style="display:none;position:absolute;left:-9999px;width:0;height:0;overflow:hidden;">`で囲み、スライドに表示されないようにする
    - **現在日を示すライン**: 現在日を示す赤いラインは`sunaneko-theme.css`で自動的に非表示になる
    - **Ganttチャートの横幅調整**: Ganttチャートを横幅いっぱいに表示するため、Ganttチャートのスライドの直前に`.mermaid`クラスに`width: 100% !important; max-width: 100% !important;`を設定したCSSを追加する（必須）
    - **その他のCSS**: Ganttチャートの横幅調整用CSS以外は、`sunaneko-theme.css`に統一されているため、スライド内にインラインCSSを追加しない
    - レイアウトは基本レイアウト（クラス指定なし）を使用。Ganttチャートは自動的に縦中央に配置される
16. **テーブルレイアウト**：
    - テーブルが5行以上の場合、**必ず** `<!-- _class: table-custom -->` を使用してカスタムテーブルスタイルを適用する
    - テーブルを強調・見やすくしたい場合も `<!-- _class: table-custom -->` を使用する

## カスタマイズ

必要に応じて、以下の要素を追加・変更できます：
- カスタムCSS（`<style>`タグ内）
  - **注意**: カスタムCSSは必要な場合のみ使用してください。HTMLとCSSを多用するとマークダウンの可読性が下がります
  - ステップ・フロー説明用レイアウトのカスタムCSS例：
    ```html
    <style>
    section.column-layout .column h2 {
        background-color: #092750ff;
        color: #FFFFFF;
        padding: 12px 20px;
        border-radius: 8px;
        margin-bottom: 16px;
        margin-top: 0;
    }
    </style>
    ```
- 数式（`$$`で囲む）
- 折りたたみ（`<details>`タグ）
- テーブル
- 引用

**重要**: マークダウンの可読性を保つため、HTMLタグやカスタムCSSは最小限に留めてください。レイアウト変更は`<!-- _class: [クラス名] -->`のコメントで行うことを優先してください。
